# 第 5 天开发推进：功能页面完善与前后端联调

## 当日目标

在 TS 暂时无法继续提供更多真实数据的情况下，优先完善系统可展示功能页面，让顶部导航中的“品牌风险、地区加盟、智能报告、项目说明”都形成独立且可说明的功能区。同时继续补充后端接口，使前端展示不只是静态页面，而是能够读取后端返回的项目进度、技术路线和数据状态。

## 已完成内容

1. 顶部导航从占位链接改为页面内真实锚点跳转，支持快速定位首页、品牌风险、地区加盟、智能报告和项目说明。
2. 新增统一的模块标题组件，保持深海军蓝、浅金色、白色卡片和冰川蓝灰背景的整体风格，使页面结构更清晰、汇报时更容易讲解。
3. “品牌风险”功能区整合搜索筛选、品牌样本卡和品牌详情，支持按品牌、股票代码、城市、品类、风险等级和使用场景筛选。
4. “地区加盟”功能区突出城市加盟环境、门店密度、竞品品牌、机会点和风险点，贴合加盟意向用户的实际需求。
5. “智能报告”功能区保留手动触发 AI 分析，展示缓存提示和 Markdown 报告预览，继续控制 DeepSeek token 消耗。
6. “项目说明”功能区展示业务场景、功能模块完成情况、技术要点、下一阶段任务和 MySQL 数据状态。
7. 后端新增 `GET /api/brand-intel/project-overview` 接口，用于返回业务场景、模块完成情况、技术路线和下一步任务。

## 技术落实

- 前端：React + TypeScript + Tailwind CSS + Lucide React + Framer Motion。
- 后端：FastAPI + Pydantic + SQLAlchemy。
- 数据库：MySQL 已完成本机库和表结构配置，前端通过数据状态接口展示数据库读取情况。
- 数据策略：数据库优先、CSV 兜底，保证真实数据未完全到位时项目仍可展示。
- AI 策略：DeepSeek 采用手动触发和缓存机制，避免自动切换品牌时重复消耗 token。

3. 增加地区多品牌对比和加盟成本对比。
4. 完善报告导出功能，后续支持 Word 或 PDF 输出。
5. 继续优化投资风险指数和加盟风险指数的评分解释。
