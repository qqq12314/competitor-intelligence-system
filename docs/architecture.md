# 系统架构说明

## 1. 改造目标

原项目侧重茶饮咖啡品牌竞品对比。本次根据最新选题报告，将项目框架升级为“茶饮咖啡小微商户轻量化信贷风控与智能经营分析系统”，保留品牌分析能力，同时新增金融风控、合同审查、商户画像和授信建议报告。

## 2. 分层设计

### 数据层

- `data/sample/brands.csv`：品牌样本库，覆盖蜜雪冰城、古茗、茶百道、喜茶、奈雪、霸王茶姬、瑞幸、库迪、幸运咖、星巴克、Manner、M Stand 等。
- `data/sample/merchants.csv`：商户经营样例库，不少于 20 条。
- `data/sample/contracts.csv`：租赁、加盟、采购合同样例。
- `data/sample/public_opinion.csv`：品牌舆情与贷后预警样例。
- `data/dictionary/risk_indicator_dictionary.csv`：不少于 30 个字段的指标字典。

### 后端层

后端采用 FastAPI，当前按业务边界拆分为：

- `brands`：品牌样本接口。
- `merchants`：商户样本接口。
- `risk`：轻量化风控评分接口。
- `contracts`：合同风险识别接口。
- `reports`：Markdown 报告生成接口。
- `dashboard`：看板汇总接口。

### AI 分析层

当前保留 DeepSeek/LangChain 入口设计，先以规则评分和 Prompt 模板完成可演示闭环，后续接入真实 DeepSeek API 后生成：

- 商户经营画像。
- 品牌风险摘要。
- 合同风险解释。
- 授信建议。
- 贷后预警摘要。
- 标准化风控报告。

### 前端层

前端采用 Vue 3、Element Plus、ECharts，页面从“竞品对比看板”调整为“金融风控看板”：

- 风险等级分布。
- 品牌样本数量。
- 高风险商户列表。
- 茶饮与咖啡品牌价格带对比。
- 七维风险雷达图。
- 合同风险类型统计。
- 舆情预警时间线。

## 3. 风控评分框架

评分维度采用七维结构：

| 维度 | 权重 | 核心字段 |
| --- | ---: | --- |
| 经营能力 | 22% | 开店时长、月流水、外卖评分、差评关键词 |
| 财务压力 | 22% | 租金占比、人工成本、采购成本、负债占比 |
| 品牌稳定性 | 14% | 品牌类别、加盟成熟度、品牌舆情 |
| 行业竞争 | 12% | 周边竞品密度、价格竞争 |
| 合同风险 | 12% | 保证金、违约责任、单方解除、预付款 |
| 舆情预警 | 10% | 负面新闻、投诉、闭店消息 |
| 渠道履约 | 8% | 外卖评分、配送评价、出餐效率 |

## 4. 业务流程

```text
商户基础信息录入
  -> 经营数据导入
  -> 合同文本上传
  -> 数据清洗
  -> 风控指标评分
  -> DeepSeek 风险解释
  -> 可视化展示
  -> 风控报告生成
  -> 人工复核与归档
```
