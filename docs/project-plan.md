# 项目具体策划案

## 1. 项目名称

基于 DeepSeek Skill 与 LangChain 的茶饮咖啡小微商户轻量化信贷风控与智能经营分析系统。

## 2. 项目背景

茶饮咖啡门店具有高频消费、流水相对清晰、外卖评价丰富、加盟主体多、价格竞争明显和经营波动较快等特点，适合作为小微信贷风控的典型课程项目场景。

传统小微信贷风控通常依赖征信、银行流水、抵押物或人工尽调，对茶饮咖啡门店的品牌竞争、门店密度、外卖评分、租金压力、加盟合同和舆情变化等非财务信号利用不足。本项目将原竞品分析方向升级为智慧金融场景下的小微商户经营贷辅助风控系统。

## 3. 一周建设目标

一周内完成一个可运行、可展示、可解释的系统原型：

1. 建立项目基础框架和开发文档。
2. 建立品牌、商户、合同、舆情样例数据。
3. 建立风控指标字典和七维评分规则。
4. 实现 FastAPI 后端核心接口。
5. 实现合同风险审查样例功能。
6. 设计 DeepSeek + LangChain 智能分析链路。
7. 实现 Vue 3 风控看板原型。
8. 生成标准化 Markdown 风控报告。
9. 整理答辩演示脚本和项目总结。

## 4. 项目框架

```text
backend/
  app/
    api/routes/          FastAPI 接口
    core/                配置
    models/              Pydantic 数据模型
    services/            业务服务

frontend/
  src/                   Vue 看板
```

整体技术流：

```text
商户与品牌数据
  -> 数据清洗和字段标准化
  -> 七维风控评分
  -> 合同风险识别
  -> DeepSeek 风险解释
  -> 前端看板展示
  -> Markdown 风控报告
```

## 5. 功能模块策划

### 5.1 商户与品牌信息库

目标：

- 建立不少于 10 个品牌样本。
- 建立不少于 20 条商户样例。
- 覆盖茶饮、咖啡、茶咖融合、加盟店、直营店、区域连锁和自营单店。

技术落实：

- Day 1 使用 CSV 存储，文件位于 `data/sample/`。
- Day 2 建立 SQLite 数据库表。
- 后端通过 `sample_loader.py` 或数据库查询服务读取数据。

关键字段：

- 品牌字段：品牌名称、品类、价格带、门店规模、加盟成熟度、舆情摘要、风险标签。
- 商户字段：城市、商圈、开店时长、客单价、外卖评分、月流水、租金占比、人工成本、采购成本、负债占比、竞品密度。

### 5.2 轻量化风控评分

目标：

- 对商户输出综合风险分、风险等级、主要风险因子、经营优势、授信建议和贷后关注点。

七维评分：

| 维度 | 权重建议 | 核心字段 |
| --- | ---: | --- |
| 经营能力 | 22% | 开店时长、月流水、外卖评分、差评关键词 |
| 财务压力 | 22% | 租金占比、人工成本、采购成本、负债占比 |
| 品牌稳定性 | 14% | 品牌梯队、加盟成熟度、品牌舆情 |
| 行业竞争 | 12% | 周边竞品密度、价格战风险 |
| 合同风险 | 12% | 保证金、违约责任、单方解除、付款条款 |
| 舆情预警 | 10% | 负面新闻、投诉、闭店消息 |
| 渠道履约 | 8% | 外卖评分、配送评价、出餐效率 |

技术落实：

- Day 3 完成 `backend/app/services/risk_scoring.py`。
- 使用 Pydantic 模型约束评分响应。
- 后续将权重抽离为配置文件，便于调整和答辩说明。

### 5.3 合同智能审查

目标：

- 对租赁、加盟、采购或供货合同中的重点条款做风险提示。

识别内容：

- 违约责任。
- 租赁期限。
- 保证金。
- 加盟费用。
- 采购付款。
- 单方解除。
- 争议解决。

技术落实：

- Day 4 先实现关键词规则识别。
- 后续将识别结果交给 DeepSeek 生成解释和修改建议。
- 输出不替代法律意见，只作为授信辅助风险提示。

### 5.4 DeepSeek 智能分析

目标：

- 生成经营画像、品牌风险摘要、合同风险解释、授信建议和贷后预警。

技术落实：

- 使用 LangChain `PromptTemplate` 组织输入。
- 使用 DeepSeek API 生成结构化 JSON。
- 使用 Pydantic 校验输出字段。
- Prompt 必须要求模型引用证据字段，数据不足时明确说明。

预期输出：

```json
{
  "business_profile": "商户经营画像",
  "brand_risk_summary": "品牌与行业风险摘要",
  "contract_risk_explanation": "合同风险解释",
  "credit_suggestion": "授信建议",
  "post_loan_warning": ["贷后关注事项"],
  "evidence_fields": ["引用的关键字段"],
  "data_limitations": "数据不足与局限说明"
}
```

### 5.5 可视化看板

目标：

- 用图表展示风险结果，支持课程演示。

页面内容：

- 品牌样本数量。
- 商户样本数量。
- 合同风险数量。
- 舆情预警数量。
- 高风险商户列表。
- 风险等级分布。
- 七维风险雷达图。
- 后续扩展品牌价格带对比和舆情时间线。

技术落实：

- Vue 3 作为前端框架。
- Element Plus 提供表格和基础组件。
- ECharts 实现雷达图、分布图和对比图。
- Axios 请求 FastAPI 接口。

### 5.6 报告生成

目标：

- 生成标准化《小微商户信贷风控分析报告》。

报告内容：

1. 商户基本信息。
2. 品牌与行业背景。
3. 经营数据摘要。
4. 风控评分结果。
5. 主要风险因子。
6. 合同风险提示。
7. 授信建议。
8. 贷后关注事项。
9. 数据来源与局限说明。

技术落实：

- Day 6 先生成 Markdown。
- 后续可使用 `python-docx` 或 PDF 工具扩展为 Word/PDF。

## 6. 接口规划

| 接口 | 方法 | 功能 |
| --- | --- | --- |
| `/api/brands` | GET | 查询品牌样本 |
| `/api/brands/{brand_id}` | GET | 查询品牌详情 |
| `/api/merchants` | GET | 查询商户样本 |
| `/api/merchants/{merchant_id}` | GET | 查询商户详情 |
| `/api/risk` | GET | 查询全部商户评分 |
| `/api/risk/{merchant_id}` | GET | 查询单商户评分 |
| `/api/contracts/review` | POST | 合同风险审查 |
| `/api/reports/{merchant_id}.md` | GET | 生成 Markdown 报告 |
| `/api/dashboard/summary` | GET | 查询看板汇总 |

## 10. 项目边界说明

本项目用于课程项目实践和原型演示，不直接输出真实金融审批结论。系统生成的风险等级和授信建议仅作为辅助分析结果，正式业务中仍需结合征信、流水原件、纳税、工商、司法、现场尽调和人工审批结论。合同审查结果仅作风险提示，不替代专业法律意见。
