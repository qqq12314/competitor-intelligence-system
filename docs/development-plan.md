# 一周开发计划

本计划按照 7 天完成课程项目可演示原型倒排。项目目标不是一次性做成完整金融生产系统，而是在一周内形成“可运行、可演示、可解释、可答辩”的轻量化原型。

## 每日计划

| 天数 | 开发主题 | 主要任务 | 当日交付物 | 验收方式 |
| --- | --- | --- | --- | --- |
| Day 1 | 项目框架与策划 | 明确选题边界、建立目录结构、准备样例 CSV、搭建 FastAPI/Vue 基础骨架、编写策划案 | README、后端骨架、前端骨架、样例数据、项目策划案 | 能看到清晰目录；后端代码可编译；文档能说明后续开发路线 |
| Day 2 | 数据层与字段标准 | 完善品牌、商户、合同、舆情样例；设计 SQLite 表；编写初始化脚本；补充字段字典 | SQLite 初始化脚本、数据模型、样例入库结果 | 能从数据库读取品牌和商户数据 |
| Day 3 | 轻量化风控评分 | 实现七维评分规则；输出综合分、风险等级、风险因子、授信建议、贷后关注点 | `/api/risk` 接口、评分服务、测试样例 | 输入商户 ID 后能返回完整评分结果 |
| Day 4 | 合同审查与 DeepSeek Prompt | 完成合同风险关键词识别；设计 DeepSeek + LangChain Prompt；预留结构化输出接口 | `/api/contracts/review`、Skill Prompt、AI 输出 schema | 合同文本能识别风险条款；Prompt 有证据字段约束 |
| Day 5 | 前端可视化看板 | 接入后端接口；完成风险分布、高风险商户列表、七维雷达图、指标卡片 | Vue 看板页面 | 页面能展示样例风险数据 |
| Day 6 | 报告生成与联调 | 完成 Markdown 报告生成；联调前后端；整理演示样例；修复主要问题 | `/api/reports/{id}.md`、样例报告、联调说明 | 能生成一份商户风控报告 |
| Day 7 | 答辩与验收材料 | 整理操作说明、演示脚本、PPT 提纲、项目总结、风险与不足 | 演示脚本、操作说明、结项总结 | 能按脚本完成 5-8 分钟演示 |

## Day 1 当前完成内容

Day 1 已完成以下内容：

- 项目从竞品分析升级为小微商户信贷风控与智能经营分析方向。
- 建立 `backend/`、`frontend/`、`data/`、`docs/`、`skills/` 目录。
- 后端建立 FastAPI 入口、配置模块、路由目录、模型目录和服务目录。
- 数据目录建立品牌、商户、合同、舆情样例，以及风控指标字典。
- 前端建立 Vue 3 + Element Plus + ECharts 看板骨架。
- Skill 目录建立 DeepSeek 风控分析 Prompt。
- 文档目录建立一周计划、Day 1 框架说明、架构说明、API 草案和项目策划案。

## 后续跟进重点

1. Day 2 优先做 SQLite 数据层，避免后续接口一直依赖 CSV。
2. Day 3 把评分规则从服务代码中拆成可配置权重，便于答辩时解释。
3. Day 4 接入 DeepSeek 前先保证 Prompt 输出 JSON schema 稳定。
4. Day 5 前端优先展示最能答辩的图表，不追求复杂页面。
5. Day 6 必须形成一份完整样例报告，作为演示闭环。
6. Day 7 将“系统仅作辅助建议，不替代人工审批和法律意见”写入答辩材料。
