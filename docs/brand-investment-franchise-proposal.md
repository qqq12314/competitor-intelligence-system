# 基于 DeepSeek Skill 与 LangChain 的茶饮咖啡品牌投资与加盟风险智能分析系统策划案

## 1. 项目概况

### 1.1 项目名称

基于 DeepSeek Skill 与 LangChain 的茶饮咖啡品牌投资与加盟风险智能分析系统

### 1.2 项目定位

本项目面向两类用户：

1. 股民与普通投资者：帮助用户了解茶饮咖啡相关品牌或上市公司的行情表现、新闻舆情、品牌扩张和投资风险。
2. 加盟意向人群：帮助用户判断某地区、某品牌是否适合加盟，分析区域竞争、品牌热度、加盟风险和未来发展潜力。

系统不直接提供确定性买卖指令，不承诺投资收益，而是提供基于公开数据的辅助分析、风险提示和决策参考。

### 1.3 项目周期

开发周期建议为 8 天至 10 天，先完成可演示原型，再逐步补充数据源、分析模型和智能报告能力。

## 2. 项目背景与意义

茶饮咖啡行业近年来发展迅速，瑞幸咖啡、星巴克、蜜雪冰城、霸王茶姬、奈雪的茶、茶百道、古茗、库迪咖啡等品牌持续扩张。对于股民来说，品牌热度、门店增长、财务表现、舆情事件和市场竞争会影响投资判断；对于加盟者来说，品牌选择、城市竞争密度、加盟费用、区域饱和度和消费者口碑会影响开店风险。

原有“小微商户信贷风控”方向更偏金融机构内部场景，需要大量商户流水、负债、合同、订单、征信等私密数据，公开爬虫难以获得。调整为“品牌投资与加盟风险分析”后，系统主要依赖公开市场行情、品牌门店分布、新闻舆情、财报公告和加盟政策，数据获取更现实，业务场景也更容易向普通用户解释。

本项目的意义在于：通过公开数据和 AI 分析能力，帮助用户快速了解茶饮咖啡品牌的投资价值、舆情风险、区域竞争情况和加盟风险，为投资者和加盟意向人群提供低门槛、可解释的辅助决策工具。

## 3. 用户需求分析

### 3.1 股民与投资者需求

股民关心的问题主要包括：

- 某个茶饮咖啡品牌或关联上市公司近期表现如何。
- 股价、涨跌幅、成交量、市值等行情指标是否异常。
- 品牌近期是否有重大新闻、负面舆情或财报变化。
- 品牌门店扩张速度和市场竞争是否会影响长期发展。
- 当前更适合关注、观望还是谨慎处理。

### 3.2 加盟意向人群需求

加盟者关心的问题主要包括：

- 某品牌在目标城市是否还有发展空间。
- 该地区同类品牌是否过多，竞争是否激烈。
- 品牌热度、加盟政策、门店密度和舆情情况如何。
- 加盟风险主要来自哪里，例如费用、选址、竞争、品牌波动或投诉纠纷。
- 是否建议在该地区加盟该品牌，后续还需要调查哪些信息。

### 3.3 共同需求

两类用户都需要：

- 搜索品牌、城市和风险等级。
- 按地区、品牌、品类、风险筛选。
- 查看品牌画像和地区市场环境。
- 获取简洁、可解释的风险结论。
- 生成一份便于阅读的智能分析报告。

## 4. 项目目标

### 4.1 总体目标

构建一个可运行、可展示、可解释的茶饮咖啡品牌投资与加盟风险智能分析系统，实现品牌搜索、地区分析、行情舆情分析、加盟风险评估、DeepSeek 智能报告生成等核心能力。

### 4.2 具体目标

1. 建立不少于 10 个茶饮咖啡品牌样本。
2. 接入品牌门店数量、城市分布、新闻舆情等公开数据。
3. 对上市或关联上市品牌补充股票行情和财务摘要。
4. 建立投资风险评分和加盟风险评分两套轻量化指标。
5. 实现品牌搜索、地区筛选、品牌筛选、风险筛选功能。
6. 实现地区加盟环境分析，包括门店密度、竞争品牌、市场热度和风险提示。
7. 实现 DeepSeek 智能分析，生成投资参考和加盟风险报告。
8. 预留 LangChain / RAG 接入能力，用于后续知识库检索和报告增强。

## 5. 数据规划

### 5.1 数据获取难度判断

与原有信贷风控方向相比，新方向的数据可获取性更高。

| 数据类型 | 获取难度 | 用途 |
| --- | --- | --- |
| 品牌基础信息 | 低 | 品牌画像、品类划分、是否上市 |
| 门店城市分布 | 中 | 地区竞争、加盟风险、市场热度 |
| 股票行情 | 中 | 投资风险、行情变化、交易热度 |
| 财报公告 | 中 | 投资价值、收入利润、经营趋势 |
| 新闻舆情 | 中 | 品牌风险、市场关注、负面事件 |
| 加盟政策 | 中 | 加盟成本、加盟门槛、合同风险 |
| 消费者评价 | 中偏高 | 口碑分析、品牌满意度 |
| 真实单店利润 | 高 | 难以公开获取，前期只做估算或提示补充 |

### 5.2 第一阶段必需数据

第一阶段建议先收集这些数据，保证系统可演示：

- 品牌名称
- 品牌类型：咖啡 / 茶饮 / 茶咖融合
- 是否上市或关联上市公司
- 股票代码或关联证券信息
- 品牌门店总数样本
- 城市门店分布
- 城市市场热度
- 地区竞争等级
- 近期新闻标题、时间、来源
- 新闻情绪：正面 / 中性 / 负面
- 品牌风险标签
- 加盟风险标签

### 5.3 第二阶段补充数据

- 股票价格、涨跌幅、成交量、市值
- 收入、净利润、门店增长率等财务摘要
- 加盟费用、保证金、管理费、选址要求
- 品牌投诉、食品安全、加盟纠纷等负面新闻
- 用户评分、口碑关键词、消费者评价摘要

### 5.4 数据来源建议

- 金融网站或金融软件公开行情页
- 上市公司公告、财报摘要、交易所公告
- 品牌官网、加盟官网、公开招商信息
- 新闻网站、财经媒体、行业资讯平台
- 地图平台或公开门店 POI 数据
- TS 已经爬取的品牌、门店、城市、新闻数据

## 6. 风险指标体系

### 6.1 投资风险指标

| 维度 | 说明 |
| --- | --- |
| 行情波动 | 股价涨跌幅、成交量、短期波动 |
| 财务表现 | 收入、利润、增长率、公告变化 |
| 品牌扩张 | 门店数量、城市覆盖、扩张速度 |
| 新闻舆情 | 正负面新闻、食品安全、加盟纠纷 |
| 行业竞争 | 同类品牌竞争、价格战、门店密度 |
| 估值风险 | 市值、估值变化、市场预期 |

输出结果：

- 投资风险分
- 投资风险等级
- 主要风险点
- 投资参考结论：关注 / 观望 / 谨慎
- 后续关注指标

### 6.2 加盟风险指标

| 维度 | 说明 |
| --- | --- |
| 地区竞争 | 目标城市门店密度、同类品牌数量 |
| 品牌热度 | 品牌新闻、搜索热度、消费者关注 |
| 加盟门槛 | 加盟费、保证金、管理费、选址要求 |
| 品牌稳定性 | 门店增长、舆情稳定性、负面事件 |
| 回本压力 | 成本投入、租金压力、市场饱和度 |
| 政策与合同风险 | 加盟条款、单方解除、费用约束 |

输出结果：

- 加盟风险分
- 加盟风险等级
- 地区竞争提示
- 品牌加盟建议：适合进入 / 谨慎进入 / 不建议进入
- 需要进一步调查的数据

## 7. 系统功能设计

### 7.1 品牌搜索与筛选

用户可以按品牌、城市、品类、风险等级进行搜索筛选。

功能包括：

- 品牌关键词搜索
- 城市/地区筛选
- 咖啡/茶饮品类筛选
- 上市/未上市筛选
- 投资风险等级筛选
- 加盟风险等级筛选

### 7.2 品牌画像

展示品牌基本信息：

- 品牌名称
- 品牌类型
- 是否上市或关联股票
- 门店规模
- 主要城市
- 品牌定位
- 近期舆情
- 风险标签

### 7.3 股票行情与投资分析

面向股民展示：

- 股票代码
- 最新价格
- 近 7 日 / 30 日涨跌幅
- 成交量变化
- 财务摘要
- 新闻舆情摘要
- 投资风险等级
- DeepSeek 投资参考解释

### 7.4 地区加盟风险分析

用户选择城市和品牌后，系统分析：

- 目标城市门店样本数量
- 该品牌在目标城市的分布情况
- 周边主要竞争品牌
- 城市市场热度
- 地区竞争等级
- 加盟风险提示
- 是否建议进入该地区

### 7.5 新闻舆情分析

系统对新闻进行分类和摘要：

- 正面新闻
- 中性新闻
- 负面新闻
- 食品安全事件
- 加盟纠纷
- 财报或公告变化
- 扩张和新品营销

DeepSeek 负责将新闻摘要转化为可读的风险解释。

### 7.6 智能报告生成

用户选择品牌和城市后，系统生成 Markdown 报告：

- 品牌基本情况
- 投资风险分析
- 地区加盟环境分析
- 竞争品牌分析
- 新闻舆情分析
- 主要风险点
- 投资与加盟建议
- 后续需要关注的数据

## 8. 技术路线

### 8.1 前端技术

前端采用 React + TypeScript + Vite + Tailwind CSS。

负责功能：

- 品牌搜索与筛选
- 品牌画像展示
- 地区加盟风险概览
- 投资风险与加盟风险卡片
- DeepSeek 分析结果展示
- Markdown 报告预览
- 响应式页面和金融风格 UI

### 8.2 后端技术

后端采用 FastAPI + Pydantic + SQLAlchemy + SQLite。

负责功能：

- 品牌数据接口
- 股票行情样例接口
- 新闻舆情接口
- 地区加盟风险接口
- 投资风险评分接口
- 加盟风险评分接口
- AI 分析接口
- 报告生成接口

### 8.3 数据层

开发阶段使用 SQLite，后续可迁移到 MySQL 或 PostgreSQL。

建议数据表：

- `brand_profiles`：品牌基础表
- `stock_market_snapshots`：股票行情快照表
- `brand_news`：品牌新闻舆情表
- `city_store_stats`：城市门店统计表
- `franchise_policies`：加盟政策表
- `brand_risk_scores`：品牌风险评分表
- `analysis_reports`：智能分析报告表

### 8.4 DeepSeek 与 LangChain

DeepSeek 负责：

- 新闻舆情摘要
- 投资风险解释
- 加盟风险解释
- 报告自然语言生成
- 用户问答回答

LangChain 后续负责：

- Prompt 模板管理
- 工具调用封装
- 结构化输出约束
- RAG 知识库检索

### 8.5 Agent 工作方式

Agent 定位为“品牌投资与加盟风险分析助手”。

工作流程：

```text
用户选择品牌和城市
  -> Agent 调用品牌画像工具
  -> 调用股票行情工具
  -> 调用城市门店和竞争分析工具
  -> 调用新闻舆情工具
  -> 调用投资/加盟风险评分工具
  -> 将摘要传给 DeepSeek
  -> 生成投资参考、加盟风险和报告
```

当前阶段可先实现规则工具链 + DeepSeek 分析接口，后续再用 LangChain 封装成正式 Agent Tools。

## 9. DeepSeek Token 控制方案

为节省 token，系统不把原始大 CSV、完整新闻全文和大量行情数据直接传给 DeepSeek。

推荐流程：

```text
原始数据
  -> 后端清洗和聚合
  -> 生成摘要 JSON
  -> DeepSeek 分析摘要
  -> 返回结构化报告
```

DeepSeek 输入示例：

```json
{
  "brand": "瑞幸咖啡",
  "target_city": "成都",
  "stock_summary": {
    "price_change_7d": "+3.2%",
    "volume_level": "较高",
    "market_signal": "短期活跃"
  },
  "store_summary": {
    "city_store_count": 260,
    "competition_level": "高竞争",
    "main_competitors": ["库迪咖啡", "星巴克", "幸运咖"]
  },
  "news_summary": {
    "positive": 6,
    "neutral": 8,
    "negative": 2,
    "main_topics": ["门店扩张", "价格竞争", "新品营销"]
  },
  "risk_score": {
    "investment_risk": "中风险",
    "franchise_risk": "中高风险"
  }
}
```

这样一次分析通常只需要几百到一两千 token，比直接传原始数据更稳定。

## 10. 系统业务流程

### 10.1 股民投资分析流程

```text
用户搜索品牌或股票
  -> 查看品牌画像和行情摘要
  -> 查看新闻舆情和风险评分
  -> 点击生成 DeepSeek 分析
  -> 获取投资风险解释和关注建议
  -> 生成品牌投资分析报告
```

### 10.2 加盟风险分析流程

```text
用户选择品牌和城市
  -> 查看城市门店密度和主要竞争品牌
  -> 查看品牌加盟政策和舆情风险
  -> 系统生成加盟风险评分
  -> DeepSeek 输出加盟风险解释
  -> 生成地区加盟风险报告
```

## 11. 项目创新点

1. 将茶饮咖啡品牌分析同时面向股民和加盟者，用户需求更清晰。
2. 用公开数据替代私密信贷数据，数据获取更现实。
3. 将 TS 爬虫数据用于城市竞争和加盟风险分析，而不是简单展示。
4. 将投资风险和加盟风险拆成两套评分体系，适配不同用户。
5. DeepSeek 只处理后端摘要数据，节省 token，提高输出稳定性。
6. 预留 LangChain / RAG / Agent 能力，方便后续扩展智能问答和报告生成。

## 14. 风险与改进方向

### 14.1 当前风险

- 部分金融行情数据可能需要稳定数据源或人工样例补充。
- 未上市品牌没有直接股票行情，只能分析品牌热度和加盟风险。
- 加盟真实盈利数据难以公开获取，需要使用估算或提示用户补充。
- 新闻舆情数据需要清洗，否则容易重复或噪声较多。
- DeepSeek 输出需要结构化约束，避免生成过长或不稳定。

### 14.2 改进方向

- 接入更稳定的行情数据源。
- 增加新闻情感分类和关键词抽取。
- 增加城市饱和度和竞品密度模型。
- 增加加盟成本与回本周期估算。
- 建立 RAG 知识库，导入加盟规则、投资风险说明和品牌资料。
- 将报告导出为 Word 或 PDF。

调整后的项目从“面向金融机构的小微信贷风控”转为“面向股民和加盟意向人群的品牌投资与加盟风险分析”，减少了对私密数据的依赖，更适合通过公开数据、爬虫数据和 AI 分析实现。

新方案保留原有搜索、地区筛选、品牌筛选、风险筛选、DeepSeek 分析和报告生成能力，同时将业务对象从商户贷款风险改为品牌投资风险和地区加盟风险。这样既保留金融分析和智能风控特色，又更贴合普通用户需求，数据获取也更可行。
