# 基于 DeepSeek Skill 与 LangChain 的茶饮咖啡品牌投资与加盟风险智能分析系统项目策划案

## 一、项目概况

### 1. 项目名称

基于 DeepSeek Skill 与 LangChain 的茶饮咖啡品牌投资与加盟风险智能分析系统。

### 2. 项目类型

本项目属于人工智能应用开发、金融数据分析、品牌风险评估与智能决策辅助系统。系统围绕茶饮咖啡行业公开数据，结合品牌行情、新闻舆情、门店分布、地区竞争、加盟政策和行业热度，为股民和加盟意向用户提供品牌投资与加盟风险分析服务。

### 3. 项目定位

项目由原先面向金融机构的小微商户信贷风控方向，调整为面向普通用户的茶饮咖啡品牌投资与加盟风险分析方向。调整后的系统更加依赖公开数据，数据获取难度更低，用户场景更明确，也更适合课程项目落地展示。

系统主要服务两类用户：

1. 股民与普通投资者：关注茶饮咖啡品牌、上市公司或关联股票的行情表现、新闻舆情、品牌扩张和投资风险。
2. 加盟意向用户：关注某地区某品牌是否适合加盟，重点分析区域竞争、品牌热度、门店密度、加盟风险和未来发展潜力。

## 二、项目背景与意义

近年来，茶饮咖啡行业持续扩张，瑞幸咖啡、星巴克、库迪咖啡、蜜雪冰城、霸王茶姬、奈雪的茶、茶百道、古茗等品牌在各城市快速布局。茶饮咖啡品牌不仅受到消费者关注，也受到股民、加盟者和行业观察者关注。

对股民而言，品牌门店扩张、财报表现、股价波动、成交量变化、新闻舆情和行业竞争都会影响投资判断。对加盟者而言，品牌热度、城市门店密度、加盟费用、同类品牌竞争、消费者口碑和地区消费能力会影响加盟决策。

原有“小微商户信贷风控”方向需要大量商户流水、负债、合同、征信等隐私数据，公开渠道难以获取，也不适合课程项目直接采集。调整后的项目主要使用公开行情、新闻、门店、品牌和加盟政策数据，数据更容易获得，业务场景更贴近普通用户需求。

本项目的意义主要体现在：

1. 面向真实用户需求，服务股民和加盟意向人群。
2. 降低数据获取难度，避免依赖隐私数据。
3. 体现 AI 在金融分析和经营决策辅助中的应用价值。
4. 通过结构化评分与 DeepSeek 解释，提高结果可读性。
5. 预留 LangChain Agent 扩展空间，支持后续智能问答和报告自动生成。

## 三、用户需求与业务场景

### 1. 股民投资分析场景

用户可以搜索某个茶饮咖啡品牌或关联股票，查看品牌基本信息、行情表现、涨跌幅、成交量、新闻舆情、门店扩张、行业竞争和投资风险等级。

典型问题包括：

1. 某品牌近期是否值得关注？
2. 该品牌或关联股票近期行情是否波动较大？
3. 品牌是否存在负面新闻或食品安全舆情？
4. 门店扩张速度是否健康？
5. 当前更适合关注、观望还是谨慎处理？

系统输出不直接给出确定性买卖指令，而是提供风险提示、趋势解释和辅助建议，避免将系统包装为绝对投资顾问。

### 2. 加盟风险分析场景

用户可以选择目标品牌和目标城市，系统分析该城市门店数量、同类竞争品牌、市场热度、消费环境、加盟政策和发展潜力。

典型问题包括：

1. 某品牌在该城市是否已经过度饱和？
2. 周边竞争品牌主要有哪些？
3. 当前地区是否适合继续加盟该品牌？
4. 加盟风险主要来自费用、选址、竞争还是品牌舆情？
5. 后续还需要线下补充调查哪些资料？

### 3. 搜索与筛选场景

系统保留品牌搜索、地区筛选、品牌筛选和风险等级筛选。用户可按品牌、城市、品类、风险等级和分析场景快速定位目标数据。

## 四、项目目标

### 1. 总体目标

构建一个可运行、可展示、可解释的茶饮咖啡品牌投资与加盟风险智能分析系统，实现品牌搜索、地区筛选、投资风险分析、加盟风险分析、新闻舆情分析和 DeepSeek 智能报告生成。

### 2. 具体目标

1. 建立不少于 10 个茶饮咖啡品牌样本。
2. 建立品牌、城市、新闻、行情、门店和加盟政策数据表。
3. 实现品牌、地区、品类和风险等级筛选。
4. 建立投资风险评分和加盟风险评分两套轻量指标体系。
5. 实现地区加盟环境分析，展示门店密度、主要竞品、市场热度和风险提示。
6. 实现 DeepSeek 手动触发分析，避免用户切换品牌时频繁消耗 token。
7. 实现 Markdown 智能报告预览，后续可扩展 Word 或 PDF 导出。
8. 预留 LangChain / RAG / Agent 扩展能力。

## 五、数据规划

### 1. 数据获取原则

项目优先使用公开、可解释、可复现的数据，避免采集个人隐私、商户流水、合同、征信和内部经营数据。

### 2. 主要数据类型

| 数据类型 | 字段示例 | 获取难度 | 用途 |
| --- | --- | --- | --- |
| 品牌基础数据 | 品牌名、品类、成立时间、门店数量、主营区域 | 低 | 品牌画像、筛选、展示 |
| 股票行情数据 | 股票代码、价格、涨跌幅、成交量、市值、更新时间 | 中 | 投资风险评分 |
| 新闻舆情数据 | 标题、来源、发布时间、情感倾向、关键词 | 中 | 舆情风险分析 |
| 门店分布数据 | 城市、门店数、商圈密度、增长趋势 | 中 | 加盟风险分析 |
| 加盟政策数据 | 加盟费、保证金、装修费、投资区间、回本周期 | 中 | 加盟成本评估 |
| 地区竞争数据 | 城市、竞品品牌、竞品门店数、商圈竞争指数 | 中 | 区域竞争分析 |
| 行业趋势数据 | 行业增速、消费热度、品类趋势 | 中 | 趋势解释与报告生成 |

### 3. 前期数据规模

前期演示阶段建议控制数据规模，避免开发成本过高：

1. 品牌样本：10 至 15 个。
2. 城市样本：8 至 12 个重点城市。
3. 新闻样本：每个品牌 5 至 10 条。
4. 行情样本：选择有上市公司或关联标的的品牌，保留最近 30 至 90 天样例数据。
5. 门店和加盟数据：优先做结构化样例数据，后续由 TS 补充爬虫数据。

### 4. 数据获取建议

TS 可优先整理公开网站、财经平台、品牌官网、加盟信息平台、新闻平台和地图类公开数据。若爬虫数据不完整，后端可以先使用样例数据跑通流程，后续按统一字段导入真实数据。

联网搜集不建议完全依赖大模型逐条检索。更合理的方式是：先由爬虫和人工整理形成结构化 CSV / JSON，再让 DeepSeek 对摘要数据进行解释，而不是让 DeepSeek 承担大规模数据采集任务。

## 六、核心指标体系

### 1. 投资风险指数

投资风险指数主要面向股民，关注品牌或关联股票的金融表现与舆情风险。

| 指标 | 权重建议 | 说明 |
| --- | --- | --- |
| 行情波动 | 25% | 近期涨跌幅、成交量变化、价格波动 |
| 舆情情感 | 25% | 正负面新闻比例、食品安全事件、品牌争议 |
| 品牌扩张 | 20% | 门店增长速度、扩张是否过快 |
| 财务与行业表现 | 15% | 财报指标、市值变化、行业景气度 |
| 竞争压力 | 15% | 竞品数量、区域竞争强度 |

系统输出结果分为低风险、中风险、较高风险和高风险，并给出关键原因。

### 2. 加盟风险指数

加盟风险指数主要面向加盟意向用户，关注地区、品牌和竞争环境。

| 指标 | 权重建议 | 说明 |
| --- | --- | --- |
| 门店密度 | 25% | 目标城市门店数量和商圈密度 |
| 竞品压力 | 25% | 同类品牌数量、头部竞品覆盖度 |
| 加盟成本 | 20% | 加盟费、装修费、保证金和投资区间 |
| 品牌热度 | 15% | 搜索热度、新闻曝光、社交讨论 |
| 地区消费潜力 | 15% | 城市消费水平、客群匹配度 |

系统输出结果包括适合关注、谨慎评估、不建议盲目进入等提示。

## 七、系统功能设计

### 1. 首页看板

首页展示系统定位、核心指标、品牌样本、地区热度、风险概览和智能分析入口。前端保持深海军蓝与浅金色金融风格，保留雪山主视觉和杂志式布局，同时将咖啡、奶茶元素低透明度放在页面边缘。

### 2. 品牌搜索与筛选

支持按品牌名、城市、品类、风险等级和用户场景筛选。用户可以快速查看目标品牌的投资风险与加盟风险摘要。

### 3. 品牌详情页

展示品牌基础信息、门店分布、行情表现、舆情列表、竞品比较、风险评分和 AI 分析按钮。

### 4. 地区加盟分析

用户选择城市和品牌后，系统展示地区门店密度、竞品品牌、市场热度、加盟成本和风险提示。

### 5. DeepSeek 智能分析

DeepSeek 分析采用手动触发机制。用户点击“生成智能分析”后，后端将结构化摘要发送给 DeepSeek，生成简短的风险解释和建议，避免切换品牌时自动调用接口造成 token 浪费。

### 6. 智能报告生成

系统支持将当前品牌和地区分析结果生成 Markdown 报告，内容包括品牌概况、数据摘要、风险评分、AI 解读、加盟建议和后续调查清单。

## 八、技术路线

### 1. 前端技术

前端使用 React、TypeScript、Tailwind CSS、Lucide React 和 Framer Motion。React 负责组件化页面开发，TypeScript 提高数据类型稳定性，Tailwind CSS 实现金融风格界面，Lucide React 提供统一图标，Framer Motion 实现克制的滚动淡入和卡片悬停动效。

前端重点由李文慧负责，主要实现：

1. 首页看板与金融风格视觉设计。
2. 品牌搜索、地区筛选、风险筛选。
3. 品牌详情、地区分析和风险卡片。
4. AI 分析结果展示和报告预览。
5. 响应式适配，保证电脑和手机端可展示。

### 2. 后端技术

后端使用 FastAPI、SQLite、SQLAlchemy 或轻量数据访问层。FastAPI 提供 REST API，SQLite 存储样例数据，后续可替换为 MySQL。后端负责数据查询、风险评分、DeepSeek 调用和报告生成。

后端重点由程浩轩负责，主要实现：

1. 品牌、城市、新闻、行情、门店和加盟政策接口。
2. 投资风险评分与加盟风险评分计算。
3. DeepSeek 手动分析接口。
4. Markdown 报告生成接口。
5. 数据导入脚本，便于 TS 后续补充 CSV / JSON。

### 3. AI 与 Agent 技术

系统使用 DeepSeek 进行风险解释和报告生成，使用 LangChain 预留 Agent 工具链。前期不做复杂 Agent 自动决策，先实现“结构化数据摘要 + DeepSeek 解释”的稳定流程。

后续 Agent 可拆分为：

1. 品牌查询 Agent：查询品牌基础数据。
2. 行情分析 Agent：分析价格和波动。
3. 舆情分析 Agent：分析新闻情感和关键词。
4. 加盟评估 Agent：分析地区竞争和加盟成本。
5. 报告生成 Agent：整合结果生成 Markdown 报告。

## 九、DeepSeek Token 控制方案

为了节省 token，系统采用以下策略：

1. 手动触发分析：用户点击按钮后才调用 DeepSeek。
2. 只传摘要数据：后端先计算评分和统计指标，再将少量结构化摘要传给模型。
3. 限制新闻数量：每次只传 Top 5 至 Top 8 条关键新闻标题和摘要。
4. 设置缓存：同一品牌、同一城市、同一日期的分析结果优先读取缓存。
5. 分层生成：列表页只显示规则评分，详情页再触发 AI 分析。
6. 控制输出长度：提示词要求输出 300 至 600 字，避免长篇报告消耗过大。
7. 报告复用：Markdown 报告可由已有分析结果拼接，不重复调用模型。

按照前期演示规模，如果每天只对少量品牌手动生成分析，token 消耗可控。系统不适合让 DeepSeek 直接联网大量搜集资料，更适合让 TS 或爬虫先整理结构化数据，再让 DeepSeek 做解释。

## 十、系统业务流程

1. TS 整理品牌、行情、新闻、门店和加盟样例数据。
2. 后端导入 SQLite 数据库，形成统一字段。
3. 用户在前端搜索品牌并选择城市。
4. 前端调用后端接口获取品牌画像、地区竞争和风险评分。
5. 用户点击“生成智能分析”。
6. 后端整理摘要并调用 DeepSeek。
7. DeepSeek 返回投资风险、加盟风险和建议。
8. 前端展示分析结果，并支持生成 Markdown 报告。

## 十三、风险与应对

| 风险 | 影响 | 应对方案 |
| --- | --- | --- |
| 数据不完整 | 评分结果不够稳定 | 先用样例数据跑通流程，后续逐步补充 |
| 股票数据接口不稳定 | 行情展示受影响 | 保留本地样例数据和更新时间字段 |
| 舆情数据质量参差 | AI 分析可能不准确 | 只传可信来源和摘要，标注数据来源 |
| DeepSeek token 消耗 | 成本不可控 | 手动触发、缓存、限制输入输出长度 |
| 项目方向变化 | 前后端需要调整 | 保留原有搜索筛选框架，只替换业务字段和指标 |

项目验收时应能够完成以下演示：

1. 打开首页，展示系统定位、品牌样本和风险指数。

7. 展示前后端接口联通和数据库样例数据。

验收重点不在于数据规模很大，而在于业务场景清晰、核心流程完整、技术路线合理、AI 调用可控、系统可运行可展示。

## 十五、总结

调整后的项目方向更加贴近用户需求，也更适合课程实训开发。系统从金融机构小微信贷风控，转向面向股民和加盟意向用户的茶饮咖啡品牌投资与加盟风险分析，既保留了金融风控和智能分析的技术特点，又降低了隐私数据采集难度。

后续开发应优先保证“可搜索、可筛选、可评分、可解释、可生成报告”的主流程跑通，再逐步补充真实数据、优化评分模型和扩展 LangChain Agent 能力。项目最终应呈现为一个专业、可信、易展示的智能分析平台原型。
